    # FAQ

    # Q：如何识别由此导致的数据不完整或者丢失？

    重新启动以后，对上次还在写入数据的bucket最新写入的文件进行扫描，不完整的标记畸形

    # Q：元数据和数据的时序以及是否实时刷盘？

    写入是先数据后元数据，删除是先元数据后数据，只有在刷盘以后才做后一步

    # Q：数据完整性扫描如何实现？

    类似Ceph的Scrub，Swift的auditor进行低频扫描，分为只扫描元数据（大小、修改时间等），和check所有数据（校验值）两种

    # Q：创建时的冲突解决方式有哪些？

    合并覆盖、重命名、报错、跳过
